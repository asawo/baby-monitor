This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Raspberry Pi baby monitor that streams webcam/microphone via RTSP ‚Üí WebRTC to a browser. Three services run on the Pi: FFmpeg (capture), mediamtx (WebRTC relay), and a Go HTTP server (serves the web UI).

## Development Commands

Requires [`just`](https://github.com/casey/just) and a configured `.env` file (copy from `.env.example`).

```sh
just build    # Cross-compile Go server for Linux ARM64
just deploy   # Build + rsync + restart HTTP service on Pi
just install  # Full setup: sync files, download mediamtx, register systemd services
just start    # Start all services on Pi
just stop     # Stop all services
just status   # Show systemd status for all three services
just logs     # Tail FFmpeg stream log
just logs-http # Tail HTTP server log
just setup    # Generate mediamtx.yml from template (substitutes PI_IP)
just sync     # Rsync source files to Pi (without binary)
```

No build/lint/test tooling beyond `go build` ‚Äî the Go server is minimal stdlib-only.

## Architecture

```
Pi:
  USB Webcam ‚Üí FFmpeg (stream.sh) ‚Üí RTSP :8554/baby
                                         ‚Üì
                                    mediamtx ‚Üí WebRTC :8889
                                         ‚Üì
                                    Go HTTP :80 ‚Üí index.html
Browser:
  index.html ‚Üí WebRTC WHEP ‚Üí mediamtx:8889/baby/whep
```

**Services** (managed by systemd):
- `stream.service` ‚Äî FFmpeg captures `/dev/baby-cam` (v4l2) + `hw:WEBCAM,0` (ALSA), outputs RTSP
- `mediamtx.service` ‚Äî Bridges RTSP to WebRTC (WHEP protocol)
- `monitor-http.service` ‚Äî Go binary serves `./public` on port 80

**Device stability**: A udev rule ([scripts/udev/99-baby-monitor.rules](scripts/udev/99-baby-monitor.rules)) creates a stable `/dev/baby-cam` symlink by matching the Logitech C270 USB IDs, so stream.service binds to `dev-baby-cam.device`.

**Web UI** ([public/index.html](public/index.html)): Single-file SPA ‚Äî no build step. Uses RTCPeerConnection for WebRTC, Web Audio API for the canvas waveform visualizer. Tuning constants at top of `<script>`:
- `SCROLL_MS` ‚Äî pixels per ms of scroll speed
- `SENSITIVITY` ‚Äî visual amplitude scale
- `YELLOW_THRESHOLD` / `RED_THRESHOLD` ‚Äî color thresholds

**Go server** ([main.go](main.go)): Trivial ‚Äî serves `./public` as static files on `:80`.

## Environment

`.env` (copied from `.env.example`) must define:
```
PI_USER, PI_HOST, REMOTE, PI_IP
```

`mediamtx.yml` is generated by `just setup` from `mediamtx.yml.example` ‚Äî do not edit the generated file directly.
